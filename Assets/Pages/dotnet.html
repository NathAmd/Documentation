.NET

Les commandes et fonctionnalités globales de .NET, ainsi qu'une section spéciale pour les structures de serveur et les threads.

Commandes .NET de Base

Les commandes .NET de base permettent de gérer les projets, les solutions et les applications .NET.

  • dotnet new: 
    
# Créer un nouveau projet .NET
dotnet new console -n MyConsoleApp
    Explication

    La commande dotnet new est utilisée pour créer un nouveau projet ou fichier .NET.

  • dotnet build: 
    
# Compiler un projet .NET
dotnet build MyConsoleApp
    Explication

    La commande dotnet build compile le projet .NET spécifié.

  • dotnet run: 
    
# Exécuter un projet .NET
dotnet run --project MyConsoleApp
    Explication

    La commande dotnet run exécute le projet .NET spécifié.

  • dotnet test: 
    
# Exécuter les tests unitaires d'un projet .NET
dotnet test MyTestProject
    Explication

    La commande dotnet test exécute les tests unitaires du projet .NET spécifié.

  • dotnet publish: 
    
# Publier un projet .NET
dotnet publish MyConsoleApp -c Release -o ./publish
    Explication

    La commande dotnet publish compile le projet et le prépare pour le déploiement.

Structures de Serveur et Threads

Les structures de serveur et les threads sont essentiels pour créer des applications .NET performantes et évolutives.

  • Création et démarrage d'un thread: 
    
    // Création et démarrage d'un thread en C#
    Thread thread = new Thread(new ThreadStart(MyMethod));
    thread.Start();
    
    void MyMethod()
    {
        // Code à exécuter dans le thread
    }
    Explication

    Vous pouvez créer et démarrer un nouveau thread en utilisant la classe Thread de System.Threading.

  • Arrêt d'un thread: 
    
    // Arrêt d'un thread en C#
    CancellationTokenSource cts = new CancellationTokenSource();
    Thread thread = new Thread(() => MyMethod(cts.Token));
    thread.Start();
    
    void MyMethod(CancellationToken token)
    {
        while (!token.IsCancellationRequested)
        {
            // Code à exécuter dans le thread
        }
    }
    
    // Pour arrêter le thread
    cts.Cancel();
    Explication

    Pour arrêter un thread de manière coopérative, utilisez CancellationToken.

  • Utilisation de Task et Task Parallel Library (TPL): 
    
    // Utilisation de Task et TPL en C#
    Task.Run(() => MyMethod());
    
    void MyMethod()
    {
        // Code à exécuter dans la tâche
    }
    Explication

    La bibliothèque TPL simplifie la programmation parallèle en utilisant des tâches au lieu de threads.

  • Programmation asynchrone: 
    
    // Exemple de programmation asynchrone en C#
    public async Task MyMethodAsync()
    {
        await Task.Delay(1000);
        Console.WriteLine("Tâche terminée");
    }
    Explication

    La programmation asynchrone permet d'améliorer la réactivité des applications en utilisant async et await.

  • Serveur-Client avec Threads: 
    
    // Exemple de serveur TCP avec threads pour chaque client
    public class Server
    {
        private TcpListener _listener;
        private bool _isRunning;
    
        public Server(int port)
        {
            _listener = new TcpListener(IPAddress.Any, port);
        }
    
        public void Start()
        {
            _listener.Start();
            _isRunning = true;
            Console.WriteLine("Serveur démarré...");
    
            // Thread pour écouter les commandes console du serveur
            Thread consoleThread = new Thread(ConsoleListener);
            consoleThread.Start();
    
            while (_isRunning)
            {
                TcpClient client = _listener.AcceptTcpClient();
                Thread clientThread = new Thread(() => HandleClient(client));
                clientThread.Start();
            }
        }
    
        private void HandleClient(TcpClient client)
        {
            NetworkStream stream = client.GetStream();
            StreamReader reader = new StreamReader(stream);
            StreamWriter writer = new StreamWriter(stream) { AutoFlush = true };
    
            writer.WriteLine("Bienvenue sur le serveur!");
    
            string message;
            while ((message = reader.ReadLine()) != null)
            {
                Console.WriteLine("Client: " + message);
                writer.WriteLine("Message reçu: " + message);
            }
    
            client.Close();
        }
    
        private void ConsoleListener()
        {
            while (_isRunning)
            {
                string command = Console.ReadLine();
                if (command.ToLower() == "exit")
                {
                    _isRunning = false;
                    _listener.Stop();
                    Console.WriteLine("Serveur arrêté.");
                }
            }
        }
    }
    Explication

    Ce code montre comment créer un serveur TCP qui utilise des threads pour gérer chaque client et un thread pour écouter les commandes console du serveur.

Utilité de .NET Vanilla

.NET Vanilla offre une base solide pour le développement d'applications robustes et performantes. Voici quelques-unes de ses principales utilités :

  • Applications Console: 
    
    // Exemple d'application console en C#
    using System;
    
    class Program
    {
        static void Main()
        {
            Console.WriteLine("Hello, World!");
        }
    }
    Explication

    .NET Vanilla permet de créer des applications console simples et efficaces pour des tâches automatisées ou des outils de ligne de commande.

  • Applications de Bureau: 
    
    // Exemple d'application Windows Forms en C#
    public class MainForm : Form
    {
        public MainForm()
        {
            Text = "Hello, Windows Forms!";
            Button button = new Button { Text = "Click Me" };
            button.Click += (sender, e) => MessageBox.Show("Button Clicked!");
            Controls.Add(button);
        }
    }
    Explication

    .NET Vanilla permet de créer des applications de bureau avec Windows Forms ou WPF (Windows Presentation Foundation).

  • Services Windows: 
    
    // Exemple de service Windows en C#
    public class MyService : ServiceBase
    {
        protected override void OnStart(string[] args)
        {
            // Code à exécuter au démarrage du service
        }
    
        protected override void OnStop()
        {
            // Code à exécuter à l'arrêt du service
        }
    }
    Explication

    .NET Vanilla permet de créer des services Windows pour exécuter des tâches en arrière-plan de manière continue.
